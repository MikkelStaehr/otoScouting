# 2026-07-08 — Opdater data fra appen (ingest-UI, fase 1)

Hidtil kørte man en dataopdatering i en terminal (`python pipeline/ingest.py`) og
kiggede i blinde. Nu er der en **"Opdater data"-service inde i Indstillinger**: én
knap genhenter alle 30 ligaer / 11.000+ spillere, mens man ser den arbejde i realtid
— og kan lukke vinduet imens.

Den gamle `/api/refresh` (kun Sofascore + 3 hardcodede FBref-ligaer fra 16-liga-
æraen, stream bundet til en åben modal) er droppet.

## Arkitektur — status-fil, ikke stream

Et fuldt run er 10-30 min, så en request-bundet stream duer ikke. I stedet:

```
POST /api/ingest/start  → spawner ingest.py DETACHED (overlever lukket fane/reload)
ingest.py               → skriver pipeline/.ingest-status.json ved hver overgang
GET  /api/ingest/status → læser filen + tjekker PID-liveness (fanger crash)
Indstillinger           → poller status hvert 1,5s, tegner fremgang
```

Fordi status ligger på disk og processen er detached: luk modalen, naviger væk, kom
tilbage om 20 min — den kører stadig og vises live. Kun ét run ad gangen (`start`
giver 409 hvis et allerede kører; PID-tjek afslører et crashet run som stoppet).

## Progress-kontrakten

`pipeline/progress.py` — en atomisk (temp+rename) statusskriver som `ingest.py` ejer.
Hvert kald er defensivt: en progress-fejl må aldrig vælte selve pipelinen.

- **steps-hovedbog:** coefficients → sofascore → transfermarkt → fbref → heatmaps →
  formations (kun de planlagte, udledt af flagene) med pending/running/done/failed.
- **leagues:** per-liga FBref-status (den er per-liga med 3 retry-passes).
- **logTail:** de sidste ~18 stdout-linjer. `run()` piper nu sub-scriptets output
  (via `_run_capture`) i stedet for at arve det — en `threading.Timer` bevarer den
  hårde timeout selv på et tavst FBref-hang.

## UI (i Indstillinger)

- **Knapper → eksisterende flags:** *Opdater alt* (`--no-spatial`) · *Kun form/xG*
  (`--sofascore-only`) · *Kun værdier* (`--tm-only`). Checkbox *inkl. heatmaps +
  formationer* → fuldt run (de to ~25-min browser-scrapes, fra som standard).
- **Live-visning:** samlet bjælke · fase-label ("FBref Eliteserien (pass 1/3)…") ·
  trin-hovedbog · per-liga FBref-grid med ⚠️ + **"igen"-retry** · log-hale · elapsed.
- Beholder den eksisterende "Data indlæst"-panel (tællere klatrer live).

Sofascore-IP-knasten (Cloudflare fra hjemme-IP ved tunge gen-kørsler) håndteres
præcis af per-liga-⚠️+retry: fejler en liga, kør kun den igen (evt. efter netskift).

## Verifikation

Uden at scrape/skrive til DB'en: Node-ruterne består idle / 400 (ukendt mode +
injektion) / 409 (concurrency) / død-pid→stoppet. `progress.py` unit-testet
(shape + overgange). En rigtig `ingest.py --fbref-only`-kørsel (bogus liga) skrev en
gyldig statusfil, fangede sub-scriptets traceback i log-halen og markerede fejl —
ingen DB/config-drift efter test.

## Tilbage (fase 2, valgfrit)

Instrumentér `fetch_sofascore.py`/`fetch_transfermarkt.py` til `142/310`-tællere for
sub-liga-granularitet. Og — hvis vi går videre med [[2026-07-08-performance-payload]]-
sporet — kan et materialiserings-trin slotte ind som sidste step i `planned`.
